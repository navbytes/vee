# Vee

A keyboard-first, plugin-extensible macOS launcher and menubar platform — a fast, local, beautiful successor to Hammerspoon.

Vee runs plugins out-of-process on JavaScriptCore, renders a declarative JSON component tree diffed with JSON Patch into native AppKit, and filters candidate lists with a native fuzzy matcher so keystrokes never cross the process boundary.

## Status

Under active construction. See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for the full implementation spec and the staged build plan.

## Repository layout

```
vee/
├─ Package.swift            # SwiftPM manifest (engine + library targets)
├─ Sources/
│  ├─ VeeProtocol/          # Shared wire contract: render tree, JSON Patch, JSON-RPC
│  ├─ VeeFuzzy/             # Native fzy-style fuzzy matcher (Part 1.5)
│  ├─ VeeCache/             # Stale-while-revalidate cache (Part 1.6)
│  ├─ VeeKeychain/          # Namespaced Keychain wrapper (cross-cutting)
│  ├─ VeeEngine/            # JSC host: VM-per-plugin, bridge, renderer, hot reload
│  ├─ VeeServices/          # Host-native providers: clipboard, app search
│  └─ VeeApp/               # AppKit launcher window + menubar shell
├─ Tests/                   # XCTest suites (TDD — written first)
├─ plugins/                 # TypeScript plugins + esbuild pipeline + SDK
└─ docs/ARCHITECTURE.md     # The spec
```

## Build & test

```sh
swift build
swift test
```

Requires macOS 14+, Swift 6+, Xcode 16+. Node 20+ is used for the plugin build pipeline (esbuild).

## Design principles

1. **Out-of-process plugins, declarative rendering** — a misbehaving plugin shows an error surface; the launcher stays up.
2. **Fetch once, filter natively** — the candidate set never re-crosses IPC on a keystroke.
3. **Stale-while-revalidate everywhere** — cached data renders instantly; refresh happens in the background.
4. **Capability-gated bridge** — every plugin declares the capabilities (network domains, fs roots, keychain namespaces) it needs; the host enforces them.
5. **Privacy is a hard requirement** — clipboard history honors `org.nspasteboard.ConcealedType`/`TransientType`/`AutoGeneratedType` and password-manager UTIs; secrets never touch disk.
